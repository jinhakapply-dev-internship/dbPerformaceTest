# dbPerformanceTest

## Branch naming rule
- main
- dev
- feature
  - feature/`기능 이름`으로 통일
  - 기능 이름은 카멜 케이스 사용
  
 
## Commit message rule
- feat :
  - 기능 구현시에 사용
- fix :
  - 기존에 구현된 기능에 logic 상의 수정이 있을 때
- style : 
  - Code Formatting, CSS 수정, 오타 수정 등 코드상의 변경이 생겼을 때
- refactor : 
  - 코드 스타일의 수정(Refactoring)되었을 때
- chore : 
  - 위 사항에 해당되지 않는 수정이 생겼을 때 (ex. 파일 위치 수정, 파일 이름 변경 등...)
